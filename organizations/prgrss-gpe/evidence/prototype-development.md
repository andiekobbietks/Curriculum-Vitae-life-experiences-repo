## Description

Developed multiple interactive prototypes for the PRGRSS mentorship app to communicate product vision, gather stakeholder feedback, and provide development specifications. Prototypes addressed key user experience challenges and introduced new features based on user research.

---

## Context

### Situation
The PRGRSS app needed significant UX improvements to better serve mentees and mentors. User feedback indicated confusion about the booking process, messaging limitations, and lack of career opportunity features.

### Task/Challenge
- Create visual prototypes that communicate product improvements clearly
- Address user pain points identified through feedback collection
- Provide development team with clear specifications
- Get buy-in from CEO and COO on proposed changes

### Action
Built multiple prototypes using Lovable:

1. **Mentor Connect Flow** - Redesigned mentor discovery and booking experience
2. **Admin Dashboard** - Improved backend management tools
3. **Mentor Connect v2** - Enhanced version with detailed messaging workflow

## Design Principles Applied

### 1. User-Centered Design

Based prototypes on actual user feedback:
- Tinder-like matching concept suggested by Eden Dion
- Job listings feature requested by multiple users
- Simplified mentor discovery based on usability issues

### 2. Progressive Disclosure

Designed interfaces that reveal complexity gradually:
- Simple initial views with detailed drill-downs
- Tooltips and guidance screens
- Step-by-step booking flows

### 3. Mobile-First

Considered both mobile and admin experiences:
> "there's going to be a mobile version build with a similar user experience as soon as I get also Hurera feedback on timeline and feasibility"

### 4. Developer-Ready

Created prototypes with implementation in mind:
> "hopefully the code should be just portable for the devs to implement"
